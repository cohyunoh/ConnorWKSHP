To All My Fellow Software Development devos, whether future or present, this PSA is being made for your benefit

This Applies to All Who Have Done:

  • A Login/Register Page
  • A Query Search Bar
  • ANY TYPE OF INPUT THAT INTERACTS WITH A SQLITE TABLE

YOU ARE AT RISK OF POSSIBLE HACKS

Let's Dive Deeper Into What These Hacks Are and How to Defend Against Them

SQLite Injection: What is It?

  • This is one of the most frequently occurring web hacks
  • It entails a hacker using web page inputs to insert malicious code in the SQL statements
  • These inputs could be login/register pages or anything that relates back to a database

SQLite Injection: Ramifications of a Hack

  • The attacker can login without having credentials
    • May be even worse if they log into an admin account
  • The attacker can insert malicious code into the database
    • They can change the contents of the database
    • Add their own account
    • Possibly add unauthorized items into a list
  • The attacker can make alterations to the availability of code by deleting tables
    • Tables can relate to other tables
    • You can lose data
    • If functions call back to that table, they will also break
  • The attacker can make the operations of the host OS fail
